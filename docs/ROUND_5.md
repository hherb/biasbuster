## What Changed in Round 5

### Training data format: markdown → JSON

The core change: `export.py` now outputs JSON after `</think>` instead of
markdown. The training output format matches what the evaluation scorer
expects via `_parse_from_json()`.

Before (V7/V8 training data):
```
</think>

## Statistical Reporting: MODERATE
- Only relative measures reported...
## Overall: MODERATE (bias probability: 40%)
```

After (V9 training data):
```
</think>

{
  "statistical_reporting": {"severity": "moderate", "relative_only": true, ...},
  "overall_severity": "moderate",
  "overall_bias_probability": 0.55,
  ...
}
```

### Boundary-grounded reasoning

Each `_build_*_reasoning()` function now adds contrastive boundary
citations: "Per boundary: MODERATE — ... Not HIGH because ..."

### Dataset growth

The V2 dataset has 1919 total examples (1533 train / 190 val / 196 test)
vs the V1 dataset's 1181 (942 train / 116 val / 123 test). The growth
comes from additional annotations accumulated in the database.

### Training: 2 epochs → 1 epoch

V9 training curves showed epoch 2 was entirely wasted — loss plateaued
at ~0.5 by step 200, gradient norms settled to ~0.5, no improvement
through steps 384-768. Config updated to 1 epoch for V10+.

## The Severity Kappa Problem

### The pattern across all rounds

| Version | Change made | Kappa | Direction |
|---------|------------|:---:|:---:|
| Baseline | (zero-shot) | 0.168 | — |
| V4 (Run 8) | Rank 32 + 2 epochs | 0.072 | ↓ |
| V6 (Run 9) | Harmony template fix | 0.017 | ↓ |
| V7 (Run 10) | Single-channel + 2 epochs | 0.095 | ↑ |
| V8 (Run 11) | JSON prompt instructions | 0.097 | ≈ |
| V9 (Run 12) | JSON training data + boundaries | 0.084 | ↓ |

**Fine-tuning has never improved severity kappa beyond baseline.** Every
change — hyperparameters, format, prompt instructions, boundary reasoning,
JSON output — moves other metrics but kappa stays in the 0.07-0.10 range,
always below baseline's 0.168.

### Root cause: ground truth label quality

The severity labels are generated by DeepSeek. The fine-tuned model learns
DeepSeek's calibration tendencies faithfully, including its inconsistencies.
The pre-trained baseline (zero-shot GPT-OSS) has its own internal
calibration that happens to be better for severity grading — fine-tuning
overwrites this with DeepSeek's noisier calibration.

Evidence:
1. Binary detection improves consistently (the model learns what IS bias)
2. Severity grading degrades consistently (the model learns DeepSeek's
   noisy severity boundaries)
3. No format/prompt change has broken the pattern across 6 versions

### What this means

The severity kappa ceiling is not a model problem, a format problem, or a
hyperparameter problem. It's a **label quality problem**. The training
labels need to be either:
- Human-reviewed for severity consistency
- Replaced with higher-quality labels (e.g., from a more capable annotator)
- Augmented with severity-contrastive examples at the moderate/high boundary

## What V9 Gets Right

Despite the kappa limitation, V9 is the most capable model overall:

1. **Calibration (0.178)**: The model's probability estimates are well-
   calibrated. When it says 0.55 probability, roughly 55% of those papers
   genuinely have bias. This is practical for downstream decision-making.

2. **Verification (0.562)**: The model consistently recommends the right
   databases. ORCID 82% (vs baseline 24%), Retraction Watch 96% (vs 24%).
   This is the fine-tuning's biggest practical win.

3. **Recall (0.971)**: Catches 97% of biased papers. For a screening tool,
   high recall with moderate precision is the right trade-off.

4. **Three significant dimension wins**: Spin, outcome reporting, and COI
   are all significantly better than baseline.

5. **100% parse success, 0 errors**: The JSON format works reliably.

### Next steps for severity kappa improvement

#### Option A: Human severity review (highest impact, most effort)

Audit the 50 most confused examples (where V9 disagrees with ground truth
by 2+ severity levels). Re-label with explicit boundary citations. This
directly fixes the noisy labels that fine-tuning amplifies.

#### Option B: Baseline-as-teacher (moderate impact, low effort)

Use the baseline's severity predictions as soft labels or as a calibration
anchor. The baseline has better severity discrimination (kappa 0.168) —
its predictions could be used to re-weight or smooth the DeepSeek labels.

#### Option C: Severity-contrastive mining (moderate impact, moderate effort)

Find pairs of papers in the training set where domain profiles are similar
but overall severity differs. Add explicit contrastive reasoning explaining
why paper A is MODERATE but paper B is HIGH. This teaches the decision
boundary directly.

#### Option D: Accept the trade-off

V9 is already a strong screening tool: high recall, good calibration,
excellent verification knowledge. Severity kappa may not be improvable
without human labels, and the practical impact of mis-grading moderate
vs high may be acceptable for the use case (the model correctly identifies
THAT bias exists, just disagrees about degree).
